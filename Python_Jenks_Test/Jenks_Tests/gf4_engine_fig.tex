\caption{The GF4-Engine front-end. (a)~A full-row Walsh--Hadamard rotation is fused
into the FP4 quantize datapath: the rotated activation feeds a per-block RMS scale
(computable in parallel from the un-rotated block, since $\lVert Hx\rVert=\lVert x\rVert$)
and the programmable codebook, then the int4 MAC and the wide accumulator that also
serves the multi-pass residual passes. (b)~Block-diagonal Hadamard collapses at
scale (opt-1.3b $A16\!\to\!7405$), so the rotation must be full-row; the Kronecker
factorization $H_M=H_G\otimes H_B$ realizes it as $G$ local $B$-wide butterflies, a
corner-turn, and $B$ cross-block $G$-wide butterflies. Every node is an
add/subtract ($\pm1$)---no multipliers---in a $\log_2 M$-stage network that reuses
one small butterfly tile. The method also runs on existing FP4 hardware (full-row
rotation in software); this engine is a minimal dedicated realization.}
\label{fig:engine}
